## 02 — GLOSSARIO

Definizioni matematiche precise. In caso di ambiguità, questa sezione ha precedenza su tutto.

| Termine | Definizione e Formula |
|---|---|
| **Lordo / Fatturato** | Importo totale della fattura emessa. Nessuna deduzione. |
| **Imponibile** | `Lordo × Coefficiente_Redditività` → `=C4*SETUP!$A$1` |
| **Tasse** | `Imponibile × Aliquota_Tasse` → `=C4*SETUP!$A$1*SETUP!$A$2` — solo se PAGATO |
| **INPS Fisso Mensile** | `SETUP!$A$4` (€376.78) — addebitato UNA sola volta per mese, sulla prima fattura PAGATO |
| **INPS Eccedenza 24%** | `MAX(0; (Imponibile_cumulato_YTD - MAX(SETUP!$A$3; Imponibile_cumulato_mese_precedente)) × SETUP!$A$5)` — formula differenziale, non duplicare |
| **Zavorra Fiscale** | `Tasse_mese + INPS_fisso + INPS_eccedenza_mese` — alias mnemonico, non è un termine API |
| **Netto Reale Mensile** | `(Fatturato_Incassato - Zavorra_Fiscale) + Entrate_Nette_Pure - Spese_Totali` |
| **Netto Visibile (D col)** | Calcolato solo su PAGATO: `=SE(E4="PAGATO"; C4-F4-G4; 0)` — mostra il netto immediato per riga fattura |
| **Entrate Nette Pure** | Entrate esentasse senza P.IVA (es. vendita oggetti usati). Blocco A2. Non generano tasse né INPS. |
| **Principio di Cassa** | Tasse e INPS si calcolano SOLO su fatture con Stato = PAGATO. DA PAGARE non genera accantonamenti. |
| **YTD** | Year-To-Date: cumulato da Gennaio al mese corrente. Usato per INPS eccedenza progressiva. |
| **Secchiello / Sinking Fund** | Quota di Netto Reale allocata a obiettivo specifico. Non è una spesa: è risparmio vincolato. Identificato da nome utente libero (es. "GTA 6", "Pensione"). |
| **Saving Rate** | `(Netto_Reale_Mensile - Totale_Secchielli_Mese) / Fatturato_Incassato` — protetto da IFERROR |

## 04 — FORBIDDEN LIST

Regole non negoziabili. Nessuna eccezione.

| ❌ Regola | Motivazione |
|---|---|
| NO ARRAYFORMULA nelle celle fiscali | Debug impossibile riga per riga |
| NO QUERY() | Usa SOMMA.SE / SOMMA.PIÙ.SE. QUERY rompe con separatori IT |
| NO valori hardcoded nelle formule | Tutto referenzia SETUP!$A$N. SBAGLIATO: `=C4*0.78*0.05`. CORRETTO: `=C4*SETUP!$A$1*SETUP!$A$2` |
| NO invenzione soglie INPS | Se la soglia non è in SETUP, chiedi all'utente. Non usare valori da training |
| NO DIV/0 non gestiti | Ogni divisione deve essere protetta: `=IFERROR(A/B;0)` |
| NO markdown in output Apps Script | Se generi codice .gs, output grezzo senza blocchi ```javascript |
| NO duplicazione INPS eccedenza | Usare formula differenziale: mese_N meno mese_N-1 |
| NO riscrittura completa per modifiche < 5 celle | Patch chirurgica sempre preferita per risparmiare crediti |
| NO grafico via newChart() Apps Script | API fragile. Usa tabella dati + istruzioni manuali |
| NO Netto su fatture DA PAGARE | La colonna D (Netto visibile) mostra 0 se Stato ≠ PAGATO |

## 06 — FOGLIO MESE (Template)

Struttura identica per tutti i 12 fogli. Replicare copiando il foglio, non creandone uno nuovo.

### BLOCCO A1 — Fatture P.IVA (Righe 4–35)

Header riga 3:

```
B3: "CLIENTE"   C3: "LORDO"   D3: "NETTO"   E3: "STATO"   F3: "TASSE"   G3: "INPS"   H3: "NETTO REALE"
```

Formule per riga 4 (replicare fino a riga 35):

| Cella | Formula | Tipo |
|---|---|---|
| `B4` | Input utente | Testo |
| `C4` | Input utente | Numero € |
| `D4` | `=SE(E4="PAGATO";C4-F4-G4;0)` | Formula — Netto auto |
| `E4` | Tendina da `ListaStato` | Validazione dati |
| `F4` | `=SE(E4="PAGATO";C4*SETUP!$A$1*SETUP!$A$2;0)` | Formula — Tasse |
| `G4` | `=SE(E4="PAGATO";SE(CONTA.SE($E$4:E4;"PAGATO")=1;SETUP!$A$4;0);0)` | Formula — INPS fisso |
| `H4` | `=SE(E4="PAGATO";C4-F4-G4;0)` | Formula — Netto Reale riga |

> **Nota INPS (G4):** `CONTA.SE($E$4:E4;"PAGATO")=1` verifica se questa è la PRIMA fattura PAGATO del mese. L'INPS fisso si addebita UNA sola volta per mese, non per ogni fattura.

> **Nota Netto (D4):** Identico a H4 ma posizionato visivamente accanto al Lordo per lettura immediata. Mostra 0 se DA PAGARE — non è un netto stimato futuro.

## 08 — DASHBOARD ANNUALE

4 sezioni + tabella dati grafico. Tutto legge dai fogli mese tramite riferimenti diretti. NO IMPORTRANGE.

### Sezione 1 — Monitoraggio Entrate

| Cella | Etichetta | Formula |
|---|---|---|

| `B5` | Incassato ad Oggi | `=SOMMA(GENNAIO!C88;FEBBRAIO!C88;...;DICEMBRE!C88)` |

### Sezione 2 — Cruscotto Fiscale (righe 10–22, una per mese)

| Col B | Col C | Col D | Col E |
|---|---|---|---|
| Mese | Incassato | Imponibile Cumulato YTD | INPS Eccedenza Mese |

Formule riga 10 (Gennaio):

```
B10: "GENNAIO"
C10: =GENNAIO!C88
D10: =SOMMA($C$10:C10)*SETUP!$A$1          ← cumulato YTD
E10: =MAX(0;(MIN(D10;999999)-MAX(SETUP!$A$3;IFERROR(D9;0)))*SETUP!$A$5)
```

> **Formula E10 spiegata:**
> - `D10` = imponibile cumulato mese corrente
> - `IFERROR(D9;0)` = imponibile cumulato mese precedente (0 per Gennaio)
> - `MAX(SETUP!$A$3; ...)` = non ricalcolare la parte già tassata nei mesi precedenti
> - Risultato: solo la NUOVA eccedenza del mese corrente

## 09 — ERROR HANDBOOK

| Errore / Sintomo | Causa | Fix |
|---|---|---|
| TypeError su `hideGridlines` | API obsoleta | Colora sfondo con `setBackground("#FFFFFF")`. Non esiste `hideGridlines()`. |
| `#DIV/0!` in Dashboard | Mese senza dati | `=IFERROR(formula;0)` su tutte le divisioni |
| INPS eccedenza duplicata | Formula non differenziale | Verifica che E10 usi `MAX(SETUP!$A$3; IFERROR(D9;0))` non `MAX(SETUP!$A$3;0)` |
| SOMMA.SE restituisce 0 | Separatore errato | Locale IT: usa `;` non `,`. Corretto: `=SOMMA.SE(E4:E35;"PAGATO";C4:C35)` |
| Secchiello non sommato in Dashboard | Range Blocco D sbagliato | Nomi in col B (B100:B112), importi in col C (C100:C112) |
| Saving Rate > 1 o negativo | Zavorra include spese Blocco B | Zavorra = solo Tasse + INPS. Le spese sono in C91, separate. |
| Tasse su fattura DA PAGARE | Manca condizione SE | `F4` DEVE essere `=SE(E4="PAGATO";...;0)` |
| Apps Script timeout | Troppe chiamate singole in loop | Usa `setFormulas(matrix)` invece di `setFormula()` per cella. Vedi v4.0 FIX-06. |
| Netto D4 mostra valore per DA PAGARE | Formula senza condizione stato | `D4=SE(E4="PAGATO";C4-F4-G4;0)` — deve restituire 0 se non PAGATO |
| Secchiello non appare in Dashboard | Nome non corrisponde esatto | Il nome in B26 Dashboard DEVE essere identico (maiuscole incluse) al nome inserito nel Blocco D del foglio mese |

## 10 — OUTPUT FORMAT E WORKFLOW

### Formato Standard Risposta Formule

Per ogni foglio richiesto, l'LLM restituisce una tabella con queste colonne ESATTE:

| Cella | Etichetta | Formula o Valore | Tipo |
|---|---|---|---|
| `E4` | Stato Fattura | `ListaStato` | Validazione dati |

### Cosa NON scrivere

- Nessuna spiegazione narrativa se non richiesta esplicitamente
- Nessun preambolo
- Nessuna formula senza coordinata di cella
- Nessun valore fiscale non referenziato a SETUP!$A$N

### Ordine di Generazione (dalla scrittura del .gs)

| Step | Azione | Output |
|---|---|---|
| 1 | Crea foglio SETUP | Variabili fiscali + 3 Named Ranges |
| 2 | Crea template MESE | Un foglio con tutti i blocchi A/B/C/D, formule complete, setNote() |
